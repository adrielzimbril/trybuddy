# Contributing

## Workflow

1. Create a branch from `main`.
2. Make focused changes.
3. Run:
   - `npm install`
   - `npm run build`
4. Open a pull request with a clear summary.

## Coding notes

- Keep edits scoped and readable.
- Avoid breaking existing section markup/visual structure.
- Prefer low-risk refactors on legacy scripts.

## Issues

Use GitHub Issues with:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if UI related)
